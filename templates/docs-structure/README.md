# プロジェクトドキュメント構造

このテンプレートは、プロジェクト開始時に`/docs`配下に標準フォルダ構造を作成するためのものです。

## セットアップ方法

### 1. テンプレートのコピー

プロジェクトルートで以下のコマンドを実行：

```bash
# 共通ルールリポジトリからテンプレートをコピー
cp -r .ai-rules/templates/docs-structure/* ./docs/
```

または、手動で各フォルダを作成：

```bash
mkdir -p docs/{consideration,todo,reference,history,requirement}
```

### 2. 各フォルダのREADMEを確認

各フォルダの`README.md`を確認し、プロジェクトに合わせてカスタマイズしてください。

## フォルダ構成

```
docs/
├── consideration/      # 要件検討
├── todo/              # 要件具体化の実装計画
├── reference/          # 参考資料管理
├── history/           # 作業記録（タイムスタンプごとのマークダウン）
└── requirement/       # 仕様書・要件定義
```

## 各フォルダの役割

詳細は各フォルダの`README.md`を参照してください。

- **consideration/**: 機能や要件を検討する段階の情報
- **todo/**: 要件を具体化し、実装計画を立てる段階の情報
- **reference/**: プロジェクトに関連する参考資料
- **history/**: タイムスタンプごとの作業記録
- **requirement/**: 仕様検討したものを具体的に仕様書・要件定義化

## 運用の流れ

1. **要件検討** (`consideration/`)
   - 機能の検討事項を記録
   - 選択肢の比較
   - 意思決定の記録

2. **実装計画** (`todo/`)
   - 要件を具体化
   - 実装タスクの分解
   - 実装順序の決定

3. **要件定義** (`requirement/`)
   - 仕様書・要件定義の作成
   - 機能要件・非機能要件の定義

4. **実装・作業記録** (`history/`)
   - 日々の作業内容の記録
   - 決定事項の記録
   - 問題と解決策の記録

5. **参考資料** (`reference/`)
   - 技術ドキュメントのリンク
   - 参考にした記事やブログ
   - 外部APIの仕様書

## 参考
